# 3D 音乐地图与深度聆听包 PRD

日期：2026-06-19  
状态：V1 草案  
产品定位：从一首歌进入一张可视化的音乐流派地图，并生成可听、可看、可继续探索的深度聆听内容。

## 1. 产品一句话

用户输入一首正在听的歌，系统在预生成的 3D 音乐流派地图中定位这首歌，点亮相关风格节点，并实时生成一个由三段电台式导听组成的 Deep Listening Pack，帮助用户理解这首歌、它连接的流派网络，以及背后的人文历史语境。

## 2. V1 核心目标

V1 不做社交，不做长期用户档案，不做完整社区。

V1 只做好一件事：让用户从一首歌进入一个立体音乐地图，并完成一次可视化、可听、可继续延展的深度聆听体验。

核心闭环：

1. 用户看到一张立体音乐地图
2. 用户输入“我正在听的歌”
3. 系统通过官方/公开结果确认歌曲
4. 系统在 3D 流派球体中定位这首歌
5. 相关流派节点、边、标签亮起
6. 用户点击“一键开启深度聆听”
7. 系统生成 3 个导听音频文件
8. 播放导听时，音乐引用卡和地图节点同步高亮
9. 导听结束后，用户选择继续深挖某个已点亮风格，或跳到完全不同的风格区域
10. 当前 session 路径保留，并可导出地图截图

## 5. 核心 UX 流程

### 5.1 地图先行

用户进入产品后，第一眼看到的是一张复杂、立体、可旋转的音乐流派地图。地图不是背景装饰，而是主产品。

地图上叠加一个输入层：

> 我正在听的歌

用户可以输入歌名、歌手或链接。

### 5.2 歌曲确认

系统通过公开来源返回候选结果。候选结果优先展示官方音频、官方 MV、可信来源结果。

用户需要确认：

> 你指的是这首歌吗？

确认后，系统进入定位流程。

### 5.3 地图定位

系统把歌曲匹配到一个或多个音乐风格节点。例如一首歌可能同时点亮：

- rock
- blues
- jazz

地图镜头拉近，对应节点和边亮起。歌曲基础信息以标签形式出现。

### 5.4 一键开启深度聆听

用户点击：

> 一键开启深度聆听

系统开始生成 Deep Listening Pack。

生成策略：

- 先生成导听 1
- 导听 2 和导听 3 后台继续生成
- 三张导听卡片分别显示状态

状态包括：

- 排队中
- 生成中
- 可播放
- 生成失败

### 5.5 三个导听卡片

生成完成后，用户看到三个并列音频卡片。

#### 导听 1：歌曲 / 艺术家深挖

聚焦这首歌本身和艺术家。

内容包括：

- 这首歌的声音特征
- 艺术家背景
- 制作特点
- 这首歌为什么会连接这些流派节点
- 本曲关键片段

#### 导听 2：流派地图扩展

聚焦上下左右的音乐拓展。

内容包括：

- 上游：这个风格从哪里来
- 下游：它后来影响了什么
- 左右：相近艺人、相邻流派、同一声音区域
- 源头片段、代表作片段、分支片段

#### 导听 3：人文历史背景

聚焦音乐背后的社会、人文、历史、宗教、地域语境。

内容包括：

- 这个声音为什么在那个时代出现
- 它和城市、族群、宗教、政治、舞厅、技术、移民的关系
- 地域变体
- 人文背景相关音乐片段

## 6. 导听内容规格

### 6.1 基本规格

- 每次生成 3 个音频文件
- 每个音频 3-5 分钟
- 总时长 9-15 分钟
- 每段音频 3-4 个音乐片段
- 每个音乐片段 10-15 秒
- 三段导听尽量不重复音乐片段
- 三段导听独立可听，但轻微连续
- 同一对主持人贯穿三段

### 6.2 主持人设定

主持人 A：懂音乐的主持人。

- 负责音乐史、流派脉络、制作分析、声音结构
- 语气成熟、清楚、有电台专业感

主持人 B：普通听众 / 感性表达型主持人。

- 负责代入用户
- 负责提出问题
- 负责自然反应和情绪承接
- 让节目不变成音乐课

整体语气：成熟电台型。

### 6.3 叙事结构

采用半固定结构。

基础结构：

1. 从用户这首歌切入
2. 播放本曲关键片段
3. 解释声音特征
4. 引出源头或相邻流派
5. 播放源头 / 代表作 / 分支片段
6. 回到音乐地图
7. 点亮相关节点
8. 电台式金句收尾
9. 给出下一步选择

收尾方式：

先用一句短的电台式收束语，再给用户选择题。

示例：

> 有些歌不是一条路的终点，而是几扇门同时打开的声音。

然后提示：

> 今天这首歌点亮了 rock、blues 和 jazz。接下来你想往哪一扇门里多走一步？

## 7. 音乐引用系统

导听中插入的每个音乐片段都是一个结构化的音乐引用对象。

### 7.1 音乐引用对象字段

```json
{
  "title": "歌曲名",
  "artist": "艺术家",
  "youtube_url": "YouTube 链接",
  "start_time": 84,
  "end_time": 98,
  "reason": "展示 funk 的 bassline 和鼓组律动",
  "genre_nodes": ["funk", "soul", "blues"],
  "map_highlight": {
    "nodes": ["funk", "soul"],
    "edges": ["soul->funk"]
  }
}
```

### 7.2 播放中的呈现

当导听播放到某个音乐片段时，播放器下方显示“本段音乐引用卡”。

引用卡包含：

- 歌曲名
- 艺术家
- YouTube 跳转链接
- 出现时间
- 引用原因
- 所属流派节点
- 在地图上的位置

同时，球体地图上对应节点和路径高亮。

### 7.3 导听结束后的引用清单

导听结束后，所有音乐引用卡保留成“本期引用音乐清单”。

用户可以：

- 回看引用歌曲
- 打开 YouTube 原曲
- 查看它在地图上的位置
- 点击任意引用歌曲，作为下一次深度聆听入口

## 8. 导听后的下一步

导听完成后，系统回到球体地图。

本次导听涵盖到的流派节点亮起。例如：

- rock
- blues
- jazz

系统问：

> 接下来你想深入了解哪一种声音？

选项包括：

- 深挖 rock
- 深挖 blues
- 深挖 jazz
- 来点完全不同的

如果用户选择已点亮风格，系统从该节点继续扩散。

如果用户选择“完全不同”，系统把用户带到球体另一端，进入一个远离当前声音区域的新风格。

## 9. Session 路径

V1 只保留当前 session 路径。

规则：

- 用户走过的节点和边在当前 session 中保留
- 用户点击引用歌曲生成下一次导听时，原路径不清空
- 新歌曲从对应节点继续延伸
- 关闭页面后可不保存
- 可使用本地缓存作为体验优化

V1 支持导出当前地图截图。

导出内容：

- 当前球体音乐地图
- 本次点亮的节点
- 本次走过的路径
- 起点歌曲
- 当前停留位置

导出风格：地图截图型，不做年度报告海报。

## 10. 风格深挖页

主地图不承载世界地图交互。

主地图只表达音乐风格之间的声音网络。

当地理维度需要出现时，它出现在风格深挖页中，作为补充模块。

例如用户深挖 Funk：

- 主球体定位 Funk 的声音关系
- 弹出 Funk 深挖页
- 深挖页展示 Funk 的谱系树
- 深挖页展示 Funk 的全球辐射小地图

小地图可以展示：

- US Funk
- Ethiopian Funk
- Japanese Funk
- Korean Funk
- Middle East Funk

小地图定位：地域传播补充视图，不是主导航。

## 11. 技术实现方案

### 11.1 总体原则

系统采用：

> 底图预生成，旅程实时生成。

用户进入产品时，音乐流派地图已经存在。

预生成内容包括：

- 流派节点
- 关系边
- 球体坐标
- 基础流派知识
- 流派代表曲目
- 风格详情骨架

用户输入歌曲后，实时生成本次聆听旅程。

实时生成内容包括：

- 歌曲匹配
- 节点点亮
- 导听路径
- 播客脚本
- 音乐片段选择
- TTS 人声
- 混音音频
- 下一步选择

### 11.2 数据来源选择

已选方案：

- 1B：Metadata-first + YouTube 播放确认
- 2B：yt-dlp + ffmpeg 作为内部原型工具
- 3B：Every Noise 参考 + 自建流派图谱
- 4B：多信号融合判断流派
- 5B：RAG 知识库 + LLM 生成导听
- 6A 修正：用户感知实时生成，底层底图预生成

### 11.3 曲库与歌曲确认

推荐来源：

- MusicBrainz：标准音乐 metadata
- Discogs：发行信息、genre/style
- Last.fm：用户标签、相似艺人
- Wikidata：结构化知识、地区、历史关系
- YouTube：搜索确认、播放入口、引用链接

YouTube 用途：

- 搜索候选
- 展示官方/公开结果
- 提供跳转链接
- 原型阶段可辅助音频处理

### 11.4 yt-dlp 与 ffmpeg

V1 原型可使用：

- `yt-dlp`：内部原型音频获取、metadata 抓取、样本分析
- `ffmpeg`：剪辑、转码、混音、ducking、响度统一

公开产品表述应保守：

- 正式环境优先使用授权音频、用户上传音频、可合法使用音频或外部播放器
- YouTube 下载不作为公开产品默认主链路

### 11.5 流派图谱

Every Noise 用作参考：

- 流派颗粒度
- 音乐地图感
- 风格相似度启发
- 分类命名参考

但 V1 需要自建 genre graph。

自建图谱包括：

- genre id
- genre name
- parent / related genres
- similarity edges
- influence edges
- fusion edges
- regional variant edges
- representative tracks
- representative artists
- era tags
- region tags
- map coordinates

### 11.6 导听生成链路

已选方案：

- 生成完整音频文件
- 先节目大纲，再逐段写双主持脚本
- 下载/获取音乐片段后混入节目
- 音乐片段全自动选择
- 分段表演式 TTS
- 自动电台混音

生成链路：

1. RAG 检索
2. 生成节目大纲
3. 确定每段音乐 cue
4. 生成双主持脚本
5. 选择音乐片段
6. 生成主持人 A/B 语音
7. 剪辑音乐片段
8. 做电台式混音
9. 导出完整音频文件
10. 生成音乐引用卡和地图高亮数据

## 14. V1 成功标准

V1 成功不以曲库规模判断，而以体验闭环判断。

最小成功标准：

- 用户能从一首歌进入 3D 音乐地图
- 用户能看到这首歌相关的流派节点被点亮
- 用户能生成 3 个不同功能的导听音频
- 用户在听导听时能看到音乐引用卡和地图同步高亮
- 用户能从导听引用歌曲继续下一次探索
- 用户能导出本次音乐地图路径截图

体验目标：

- 用户觉得这不是普通推荐器
- 用户觉得自己在音乐地图里行走
- 用户能理解一首歌背后的流派、源头、分支和文化语境
- 用户愿意从引用音乐继续点下去
